## 一、产品定位

### 1.1 一句话

**每天 1 个穿搭微调。不教你买什么，教你「怎么穿」。**

### 1.2 核心主张

市面上的穿搭产品都在解决一个问题：「你应该穿什么衣服」。我们解决另一个问题：「**你已经有衣服了，怎么把它穿好看**」。

同样的白衬衫 + 牛仔裤——塞不塞、卷不卷、解几颗扣——是路人还是博主的差距。这些微小的穿搭技巧，才是普通用户的时尚盲区。我们在做的就是：**时尚最后一公里**。

### 1.3 目标用户

- 已经会基本穿搭，但感觉"差一口气"的人
- 照着博主买同款，穿上就是不对味的人
- 每天早上站在衣柜前纠结"今天穿什么"的人
- 想提升穿搭水平，但没有精力和预算系统学习的人

### 1.4 使用门槛

- ❌ 不需要录入衣橱
- ❌ 不需要填写问卷
- ❌ 不需要关注博主
- ✅ 首次：拍 3 张身形照（60 秒）
- ✅ 每天：拍 1 张穿搭照（15 秒）

## 二、核心产品逻辑

### 2.1 三层递进架构

```
┌─────────────────────────────────────────────┐
│                                             │
│  Layer 1：学会穿                              │
│  拍照 → AI审稿 → 微调建议 → 前后对比 → 打卡     │
│                                             │
│  用户获得：可以带走的穿搭技巧 + 肉眼可见的进步    │
│  产品获得：身形数据 · 衣橱数据 · 风格偏好 · 行为数据│
│  ⏱️ Day 1 ~ Day 30                          │
│                                             │
│                  ↓ 数据触达，AI主动邀请       │
│                                             │
│  Layer 2：帮你穿                              │
│  AI比你更了解你的衣橱和风格 → 每日3套方案         │
│  左滑右滑 → 数据回流 → 越来越准                 │
│  ⏱️ Day 30+                                 │
│                                             │
│                  ↓ 衣橱数字化完成 + 风格DNA形成 │
│                                             │
│  Layer 3：聪明买                              │
│  AI发现衣橱缺口 →「你缺这件来完成你的风格」       │
│  精准单品推荐，基于真实需求而非猜你喜欢           │
│  ⏱️ Day 60+                                 │
│                                             │
└─────────────────────────────────────────────┘
```

### 2.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **渐进式数据采集** | 不要求用户一次性录入，数据在日常使用中自然积累 |
| **先给价值再取数据** | 每个交互都给用户即时可见的价值，然后默默记录数据 |
| **被动衣橱数字化** | 用户每天拍照，AI 自动提取单品，衣橱在后台自然建成 |
| **AI 是审稿人，不是作者** | AI 不否定用户的穿搭，而是在已有基础上优化 |
| **技巧可携带** | 用户学到的是自己的本事，离开 App 也能用 |

### 3.2 AI 审稿引擎

AI 执行的是一个穿搭审稿人的完整思维链：

```
┌────────────────────────────────────────────┐
│ 1. 解析今日穿搭                              │
│    → 品类识别、颜色提取、面料判断、版型分析      │
│    → 识别用户今天穿了什么、怎么穿的              │
│                                            │
│ 2. 对照用户身形数据                           │
│    → 肩宽/腰线/腿长比例/头身比                 │
│    → 当前穿搭对身形的修饰效果评估               │
│                                            │
│ 3. 对照用户风格目标                           │
│    → 用户选择的风格方向（可多选、可修改）         │
│    → 当前穿搭与目标风格的契合度评估              │
│                                            │
│ 4. 对照用户技巧历史                           │
│    → 已掌握的技巧列表                         │
│    → 当前学习中的技巧                         │
│    → 历史微调接受率                           │
│                                            │
│ 5. 选择最优微调建议                           │
│    → 从技巧库中筛选候选（匹配当前穿搭品类）       │
│    → 按"效果显著度 × 执行容易度 × 水平匹配度"排序 │
│    → 选择排名最高的 1 个建议                   │
│    → 附带可执行的具体指导                      │
│                                            │
│ 6. 生成改进效果图                             │
│    → 用户原图 + 身形模型 + 微调参数             │
│    → Seedream 生成 AFTER 效果图               │
│                                            │
│ 7. 估算提升量化                               │
│    → 清爽度 / 比例感 / 风格完成度的预估变化      │
└────────────────────────────────────────────┘
```

### 3.3 技巧知识库结构

```
技巧库 🌳

🌱 基础篇：衣服怎么穿（门槛：第1天起）
├─ 塞法系列
│   ├─ 全塞（Tuck All）：全塞进下装，拉出少许松量
│   ├─ 法式半塞（French Tuck）：只塞前摆，后摆自然垂落
│   └─ 随意前塞：只塞正前方一小块，其余自然堆叠
├─ 裤脚系列
│   ├─ 单折卷边：向外翻折一次，宽约2-3cm，整洁感
│   ├─ 双折细卷：Pinroll，向内折细，锥形裤专用
│   └─ 随意堆叠：不卷，自然堆叠在鞋面上，街头感
├─ 袖口系列
│   ├─ 基础翻折：解开袖口，外翻两圈，露出手腕
│   ├─ 推肘堆叠：长袖推到肘部，制造随意层次
│   └─ 卷袖至肘：规整地卷至肘下，工装风格
├─ 领口系列
│   ├─ 扣子法则：衬衫最上两颗不扣，Polo最上一颗不扣
│   ├─ 立领造型：外套领立起，防风+造型
│   └─ 叠穿领口：内层领口高出外层的分寸
└─ 腰带系列
    ├─ 颜色呼应：腰带颜色与鞋/包/表的呼应规则
    ├─ 随意垂坠：不穿过扣环，腰带随意垂下
    └─ 无腰带造型：什么时候不需要系腰带

🌿 进阶篇：比例怎么调（门槛：掌握3个基础技巧）
├─ 上短下长
│   ├─ 视觉高腰：通过塞衣/短款/高腰裤拉高腰线
│   └─ 内搭短外搭长：叠穿中的比例控制
├─ 层次叠穿
│   ├─ 三露法则：领口露·下摆露·袖口露
│   └─ 长度递进：内→中→外 长度依次递增
├─ 露肤分寸
│   ├─ 露踝法则：九分裤/卷边，露出最细的脚踝
│   ├─ 露腕法则：袖口处理，露出手腕
│   └─ 不露的时机：正式场合/寒冷天气的替代方案
└─ 松紧平衡
    ├─ 上宽下窄：宽松上装+修身下装
    ├─ 上窄下宽：修身/塞衣上装+宽松下装
    └─ 全宽收腰：全宽松+腰带收腰

🌳 高级篇：风格怎么塑（门槛：掌握8个技巧+Layer 2解锁）
├─ 色彩桥接
│   └─ 用一件中间色单品，连接上下身对比色
├─ 材质对冲
│   └─ 硬+软·光+糙·厚+薄的对冲搭配法则
├─ 风格混搭
│   └─ 正式+休闲·复古+现代·运动+优雅
├─ 配饰点睛
│   └─ 项链/手表/戒指/包/帽子/袜子的位置与呼应
└─ 造型破格
    └─ 故意不塞·故意不对称·故意堆叠——知道规则后打破它

🎯 大师篇：自己怎么创（门槛：掌握15个技巧+90天+）
├─ 穿搭签名：建立你独有的穿搭印记
├─ 色彩人格：找到你的专属色系组合
├─ 趋势翻译：把秀场趋势变成你的日常版本
└─ 规则即兴：在了解规则的基础上自由创作
```

## 四、Layer 2 详细设计：帮你穿

### 4.1 触发机制

**不是固定时间解锁，是数据条件解锁**：

```
系统在后台持续检测以下条件：

✅ 累计 20+ 天有效穿搭照片
✅ 身形模型置信度 > 85%
✅ 风格偏好方向已收敛（Top1 风格连续14天未变）
✅ 已识别 15+ 件常穿单品
✅ 微调接受率 > 60%

→ 全部满足时，推送通知：

  "这 20 天，你从穿搭 42 分进步到了 68 分。
   我觉得我已经足够了解你的风格了。
   
   从明天开始，让我帮你挑选穿搭？
   
   [好，试试看]  [我还是自己搭]"
```

### 4.3 设计要点

- **每天 3 套，不是无限滑动**。AI 的价值在于帮你决策，而不是给你无限选项让你继续纠结
- **第 3 套永远是"舒适区边缘"**——保持探索性，防止用户陷入穿搭惯性
- **右滑 = 采纳** → 明早推送提醒 → 第二天晚上弹窗"今天穿了吗？"→ 记录下来
- **左滑 = 不采纳** → 同风格换另一套（不是随机换，是了解你为什么不喜欢）
- **通过右滑左手滑**的过程继续学习用户偏好，推荐越来越准

## 五、Layer 3 详细设计：聪明买

### 5.1 触发场景

**场景一：衣橱结构缺口**
```
AI 分析你的衣橱后：

"你的 Clean Fit 骨架已经有了：
 ✅ 基础T恤 4件
 ✅ 基础衬衫 2件
 ✅ 牛仔裤 2条
 ✅ 帆布鞋 1双
 ❌ 少一件轻薄外套（如灰色轻质西装）

加上它，你现有的 30+ 件衣服可以多出 15 套新组合。"
```

**场景二：技巧升级需要**
```
"下个月你可以学习【色彩桥接】了。
但要练好这个技巧，你需要一条彩色腰带或一双彩色袜子。
目前你的衣橱里没有这类单品。"
```

**场景三：单品退役提醒**
```
"你的白T恤已经穿了约 60 次。
领口开始变形的迹象可能影响整体质感。
以下是 3 个相似但略有升级的替代品。"
```

### 5.2 推荐逻辑

| 传统电商 | 我们 |
|---------|------|
| "你可能喜欢" | "你缺这件来完成你的风格" |
| 基于浏览/购买历史 | 基于衣橱结构分析 + 技巧升级路径 |
| 推荐同类商品 | 推荐互补单品 |
| 鼓励冲动消费 | 鼓励审慎填空 |

## 六、数据架构

### 6.1 用户数据模型（渐进式构建）

```
用户数据（随时间自然丰富）

Day 1 ─ 首次设置
├─ 身形照片（3张）
├─ 身形分析结果（体型/肤色/比例）
└─ 风格目标（1-2个方向）

Day 1-6 ─ 每日拍照积累
├─ 每日穿搭照片
├─ AI穿搭分析记录（穿了什么·怎么评价）
├─ 微调记录（建议了什么·接受了吗·执行了吗）
└─ 单品识别结果（品类/颜色/面料/版型）

Day 7+ ─ 开始形成偏好
├─ 风格偏好收敛数据
├─ 颜色偏好统计
├─ 技巧掌握清单
├─ 穿搭评分曲线
└─ 连续打卡天数

Day 30+ ─ Layer 2 启用
├─ 被动建立的虚拟衣橱（30+件单品）
├─ 风格 DNA（固化偏好 + 探索边界）
├─ 穿搭方案采纳率
└─ 单品使用频率统计

Day 60+ ─ Layer 3 启用
├─ 衣橱结构分析报告
├─ 缺口识别
└─ 购买推荐历史

全程 ─ 行为数据
├─ 打开时间/频率/时长
├─ 哪些功能用得多/少
├─ 在哪里卡住/退出
└─ 分享/推荐行为
```

### 6.2 隐私设计

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| 身形照片 | 设备端优先 | 仅在本地用于身形建模，可选云端备份 |
| 每日穿搭照片 | 云端加密 | 用于 AI 分析和历史对比 |
| 身形参数 | 云端 | 体型/肤色/比例等数值化数据 |
| 衣橱数据 | 云端 | 自动识别的单品信息 |
| 行为数据 | 云端脱敏 | 用于改进产品，不关联个人身份 |

## 八、MVP 范围

### 8.1 Phase 1：核心体验闭环（6-8 周）

**目标**：验证"拍照→微调→打卡"这个核心循环是否有用户价值

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 身形拍照 + AI 分析 | P0 | 首次设置的 3 张照片 + 身形报告 |
| 每日穿搭拍照 | P0 | 拍照 → AI 分析 → 微调建议 |
| 效果图生成 | P0 | BEFORE/AFTER 对比图 |
| 打卡系统 | P0 | 每日打卡 + 连续天数 |
| 基础技巧库（15个） | P0 | 基础篇全部 15 个技巧 |
| 个人主页 | P1 | 打卡天数 + 掌握技巧 + 成就 |
| 周报 | P1 | 每周穿搭进步报告 |
| 技巧卡（不拍照模式） | P1 | 今天不想拍照时的替代体验 |

**不含**：Layer 2/3、高级技巧库、风格方向选择、社区功能

### 8.2 Phase 2：Layer 2 + 进阶技巧（+6 周）

- Layer 2 推荐引擎（数据条件触发）
- 被动衣橱数字化（从历史照片提取单品）
- 进阶技巧库（20 个）
- 风格方向选择与收敛

### 8.3 Phase 3：Layer 3 + 变现（+4 周）

- 衣橱结构分析与购买推荐
- 会员体系上线
- 高级技巧库
