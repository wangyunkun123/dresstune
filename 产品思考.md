# 产品思考

> 创建于 2026-06-29 · 讨论者：kun + Claude

---

## 问题的起源

穿搭推荐项目的核心功能已基本完成。但我意识到一个非常严重的问题：**衣服上传这一步骤会打消大部分用户的兴趣**。

### 当前两种录入方式

| 方式 | 问题 |
|------|------|
| 单件上传（干净背景拍单品） | 太慢，女性用户 100+ 件衣服，逐件上传 5-10 小时 |
| 拍上身照 AI 识别 | 不稳定，无法识别剪裁、面料、背面细节，导致推荐失误 |

### 当前录入流程的技术细节

- 前端：拍照/相册 → Canvas 压缩到 720px/≤500KB（Tailscale Funnel 限制）→ POST /api/wardrobe/add → 轮询 /api/task/{tid} → 审核界面 → 确认入库
- 后端：皮肤检测 → SegFormer 单品分割 → Seedream 白底重构 → 豆包 VLM 分析 → 品类 fuzzy 匹配 → ID 分配 → EXIF 校正 → AI 方向检测 → rembg 抠图 → 缩略图生成 → 标签 JSON → 服装档案.md 更新
- 平均每件耗时 **3-4 分钟**，100 件 = 5-7 小时

---

## 核心洞察

**如果录入是瓶颈，就不要让用户录入。而不是想办法让录入更简单。**

就像 iPhone 用触摸屏取代了键盘——不是让键盘更好用，而是消灭了键盘。

## 产品方向讨论

我们讨论了三个方向：

### 方向 A：衣橱数字化路线（解决录入问题）

**核心命题**：用户有衣服，AI 帮忙搭。用技术创新把录入做到极致简单。

**推荐方案：三层漏斗**
```
第一层（秒级）：电商订单导入 + 品牌数据库匹配（覆盖 30-50%）
第二层（分钟级）：衣柜视频扫描，一次性批量处理（覆盖剩余 50-70%）
第三层（持续）：拍照补录 + 购物时自动添加（零摩擦）
```

**优势**：已有资产全部复用（99 风格 + 评分引擎 + 生图管线 + 抠图处理），护城河深。
**劣势**：即使做到极致仍需 5-10 分钟初始设置；中国市场电商 API 封闭；用户基数天花板有限；仍然是"维生素"产品（nice to have）。
**变现**：订阅制 $5-10/月。
**TAM**：全球 5000 万 - 1 亿 DAU 潜力。

### 方向 B：风格引擎路线（无需衣橱数据）

**核心命题**：用户不需要告诉 AI 自己有什么，AI 告诉用户应该穿成什么样。

**产品形态**：用户输入身形 + 风格偏好 → AI 推荐穿搭方案 + 效果图 + 购买链接 → 用户去买或在自己衣柜里找类似的。

**优势**：零上载摩擦（30 秒完成设置）；直接变现（电商分佣）；病毒传播（效果图天然适合社交分享）；市场规模巨大（中国 3 亿网购服装用户）。
**劣势**：放弃了"真实衣服"的独特价值；与小红书/抖音博主差异化不足；电商分佣在中国利润率极低；效果图理想化 → 退货率高 → 信任崩塌；竞争激烈（大平台都在做）。
**变现**：电商分佣（薄利多销）+ 品牌广告。
**TAM**：中国 3 亿网购服装用户。

### 方向 C：混合路线（推荐方案）

**核心命题**：用方向 B 的低门槛获取用户，用方向 A 的深度功能留住用户并变现。

**用户旅程**：
- Day 1（2 分钟）：输入身形 + 选 3 种风格 → 立即获得 AI 穿搭推荐 + 效果图 + 购买链接
- Week 1：每日推送天气穿搭；"我有类似的"→ 拍照识别 → 自动建虚拟衣橱；"我想买"→ 一键购买；评分反馈 → AI 学习
- Month 1：已不知不觉录入 30+ 件 → 开启衣橱模式 → 精准搭配 → 衣橱分析 → 付费转化

**产品矩阵**：

| 层级 | 功能 | 数据需求 | 商业模式 |
|------|------|---------|---------|
| 🆓 免费层 | 每日 AI 穿搭推荐 + 效果图 + 购买链接 | 身形 + 风格偏好 | 电商分佣 |
| 🥈 基础会员 | 衣橱分析 + 真实衣服搭配 + 利用率追踪 | + 衣橱数据 | 订阅 $5/月 |
| 🥇 高级会员 | 私人 AI 造型师 + 衣橱规划 + 趋势报告 | + 深度偏好 | 订阅 $15/月 |
| 👑 品牌合作 | 趋势数据 + 用户风格画像 + 精准推荐 | 脱敏聚合数据 | B2B SaaS |

**为什么这是最优路线**：
1. 解决了"鸡生蛋蛋生鸡"——先产生价值再获取数据，而非反过来
2. 符合行为心理学——用户是因为"想知道明天穿什么"而用 App，不是"应该管理衣橱"
3. 市场时机正确——2026 年 AI 生成内容接受度空前高，但还没有产品把这个闭环做好

**关键调整**：
- 衣橱数据从"前提条件"降级为"增值功能"
- "被动录入"取代"主动录入"（用户说"我有类似的"→ 自动入库）
- 从"管理过去"转向"设计未来"

## 市场数据

### 核心数据
- ~80% 的衣服常年不穿（ThredUp 2024）
- 大部分用户第一周就放弃衣橱 App
- 一款 15 万用户、运营 5 年的衣橱 App 最终关闭
- 女性平均拥有 100-150 件衣服

### 衣橱 App 五大死因
1. 🧱 **数据录入墙** — 没体验到价值就在设置阶段放弃
2. 🐌 **价值复利太慢** — 录入 10 件几乎无法给出有意义的推荐
3. 🔄 **行为改变太难** — 用户不会突然开始用 App 规划穿搭
4. 💸 **付费意愿极低** — 衣橱管理是"维生素"不是"止痛药"
5. 📉 **衣橱更新滞后** — 买新不录、删旧不删，数据越来越脏

### 全球在线时尚市场
- 全球：**$1.2 万亿**（2026）
- 中国：**$4000 亿**
- AI in Fashion 市场预计 2026-2030 年 CAGR 40%+

## 第二轮思考：放弃 A/B/C，重新出发

> 2026-06-29 续

### 为什么三个方向都不满意

kun 对三个方向的反馈：

| 方向 | 问题 |
|------|------|
| A | 难以解决的衣橱上传难题 |
| B | 同质化内容太多，大厂已在做，发展空间小 |
| C | 融合后产品太不聚焦，没亮点，难吸引用户 |

### 重新定义问题

之前一直在问："**如何让用户把衣服录入系统？**"

真正应该问的是："**用户每天早上最焦虑的是什么？**" —— **"我今天穿什么？"**

这不是衣橱管理问题、不是风格教育问题、不是购物推荐问题。这是一个**每日决策问题**。

最好的日用决策产品参考：
- 🌤️ 天气 App：打开 → 看到天气 → 关掉。零设置。
- 🗺️ 地图 App：输入目的地 → 路线 → 出发。不需要提前录入所有路。
- 🦉 多邻国：打开 → 完成今日任务 → 关掉。连胜打卡。

它们的共同特点：**每天 30 秒解决一个问题，越用越懂你，不需要"设置"。**

## 第二轮方案探索

### 方案一：📸「今日穿搭日记」— 拍照即输入

每天出门前拍一张穿搭，AI 分析 + 打分 + 给一条改进建议。越拍越懂你。

```
Day 1:  拍照 → AI: "不错！白T+牛仔裤是万能公式。
        明天试试把白T换成条纹衬衫，会更有层次感。"
        
Day 30: 拍照 → AI: "你的风格是【Clean Fit 简约休闲】，
        和东京的 xxx 博主风格相似。
        明天 28°C 有雨，建议：浅灰Polo + 深蓝九分裤 + 白色帆布鞋"
```

**用户操作**：打开 → 拍照 → 看到反馈。每天 15 秒。

**kun 的反馈**：方向类似想要的样子，但不能成为真正的产品长时间留住用户——用户完全可以直接打开 AI 对话应用来解决，没必要单独做一款产品。

**教训**：如果产品核心只是"拍照→AI点评"，ChatGPT/豆包/Claude App 随时可以替代。必须做通用 AI 做不到的事。

### 方案 A（新版）：🔥「风格克隆」— 看见喜欢的，秒变自己的

刷小红书/抖音看到喜欢的穿搭 → 截图 → AI 在用户身形上重现这个风格 → 告诉用户怎么穿。

**kun 的反馈**：还是需要有用户的数据，建立衣橱才能实现效果，还是难以解决的老问题。

### 方案 B（新版）：🎮「穿搭挑战」— 每天一个创意约束

Wordle 式的每日穿搭挑战。AI 每天给一个创意限制（如"禁止黑白灰""单色调穿搭"），用户在限制下穿搭，拍照打卡。

**kun 的反馈**：太接近小红书模式了，没有突破点。为什么不直接在小红书上做挑战呢？

---

### 方案 C（新版）：🪞「AR 试穿镜」— 站在镜子前，看到明天的自己

对着镜子拍视频，AI 在身形上叠加明天的穿搭效果。

**kun 的反馈**：目前技术达不到，没法实现。

---

### 方案 D：💡「每天一套」— 穿搭版 Tinder

每天推一套穿搭效果图，用户只需左滑右滑。滑了 30 天，AI 比用户还懂用户适合穿什么。

**kun 的反馈**：产品思路有趣，用陌生人交友的形式探索穿搭风格体验新奇，但没落到真实穿搭需求上，纯粹为了好玩。

---

### 方案 E：💡「风格补给站」— 每天一个微行动

每天给一个具体的、可执行的穿搭微动作。像健身教练一样——不是看冠军身材，而是"今天做 10 个俯卧撑"。

**kun 的反馈**：核心逻辑很认可。市面上所有穿搭应用都在教用户穿什么衣服，而不是教用户如何把衣服穿好。这很重要。

## 核心洞察：时尚最后一公里

### 关键转折点

kun 提出：

> "市面上的所有穿搭应用都是在教你穿什么衣服，而不是教你如何把衣服穿好。这就像是有一个经验丰富的穿搭师，看到你今天的穿搭，不是直接否定这套穿搭，二是进行改进。就如同同样一条丝巾，可以系在脖子里、绑到胳膊上、系到腰带上，这一点点的改变可能就是决定时尚的关键，这些才属于普通用户的时尚盲区。"

> "来选择我们产品的用户肯定都懂基本穿搭，但是离时尚还有差距，我们要解决的就是这时尚最后一公里的问题。"

### 核心洞察重述

```
普通穿搭 App：  "你应该穿这件白衬衫 + 那条黑裤子"  ← 教用户买什么
我们的产品：     "白衬衫解开两颗扣 + 袖口卷两圈"    ← 教用户怎么穿
```

用户不需要被告知"买什么"。用户家里有衣服。用户需要的是——**同一个东西，怎么穿出不同的感觉**。

## 第三轮：穿搭技巧教练方向

### 三个产品概念

#### 概念一：「微调」— 拍一张，改一处

用户拍今天的穿搭 → AI 分析 → 给一个具体的微调（如"衬衫下摆只塞前面一半"）→ 生成改进效果图 → 用户执行后拍照对比 → 打卡

#### 概念二：「点睛」— 穿搭界每日一练

把穿搭技巧拆解成可执行的每日任务，像健身教练一样逐个突破。技巧分四个阶段：基础篇（衣服怎么穿）→ 进阶篇（比例怎么调）→ 高级篇（风格怎么塑）→ 大师篇（自己怎么创）

#### 概念三：「穿对」— 同一套衣服，两种穿法

用户拍照 → AI 生成"同样衣服但穿法不同"的对比图 → 用户直观感受技巧的效果差异

**结论**：三个概念不是互斥的，是一个产品的不同切面。

## 产品逻辑定调：三层递进

kun 提出：

> "真正的产品逻辑应该是，指导用户怎么穿好看，搜集到用户足够的数据后开始推荐用户穿什么，再之后才是指导用户该怎么买，层层递进，逐步提高用户使用率和留存率。"

> "指导用户怎么穿好看，采用用户拍照，AI分析思考给出建议，并给出改进后的效果图，供用户参考去做。后续的AI推荐穿搭有足够多的用户数据后，可以做成类似于穿搭版 Tinder，这时候也有了用户的服装数据，可以进一步去推荐购买了。"

> "最终目的是要做一个轻量化但是能长时间留住用户的应用，让用户使用起来无负担。"

### 三层递进架构

```
Layer 1：学会穿
拍照 → AI微调 → 前后对比 → 打卡
用户获得：穿搭技巧 + 每天肉眼可见的进步
产品获得：身形数据 + 衣橱数据 + 风格偏好 + 行为数据
周期：Day 1 ~ Day 30

        ↓ 数据够了，自然过渡

Layer 2：帮你穿
AI知道用户有什么 + 喜欢什么 → 每天推送方案
左滑右滑 → AI越来越懂用户
周期：Day 30+

        ↓ 衣橱数字化完成 + 风格DNA形成

Layer 3：聪明买
AI发现衣橱缺口 → 精准推荐单品
不是"猜你喜欢"，而是"你缺这件来完成你的风格"
周期：Day 60+
```

### 产品定位一句话

> 每天 1 个穿搭微调。不教你买什么，教你**怎么穿**。

### 为什么这个产品 AI 对话 App 做不到

| | 我们 | ChatGPT |
|---|---|---|
| 持续追踪你的穿搭 | ✅ 每张照片都在积累 | ❌ 每次对话从零开始 |
| 在你身形上生效果图 | ✅ 专用模型 | ❌ 没这个能力 |
| 结构化技巧课程 | ✅ 100+技巧 × 分级 | ❌ 碎片化建议 |
| 知道你的衣橱 | ✅ 被动积累 | ❌ 不知道 |
| 知道你的进步轨迹 | ✅ 历史对比 | ❌ 只有当前对话 |
| 习惯养成引擎 | ✅ 打卡/连胜/成就 | ❌ 没有 |
