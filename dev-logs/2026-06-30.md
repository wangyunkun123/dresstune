### Phase 3：匹配引擎升级（15 因素评分）

**`core/skill_matcher.py`** — 核心匹配引擎

升级后的评分公式（从 4 因素 → 15 因素）：

```
score = body_type_score(0-10)            # 身形类型
      + body_modifier_bonus(-5~+5)       # 🆕 长躯干/短腿/粗腰等
      + NO_relevance(0~+5)               # Checklist 问题关联
      + scenario_match(0~+3)             # 场景匹配
      + style_direction_bonus(0~+5)      # 🆕 风格方向匹配
      + aesthetics_synergy(0~+5)         # 🆕 美学维度协同
      + weather_relevance(0~+3)          # 🆕 天气适配
      + estimated_impact(0~+5)           # 🆕 预期效果
      + difficulty_bonus(0~3)            # 难度奖励
      - style_anti_synergy(-5)           # 🆕 风格反模式惩罚
      - techniques_inferred(-3)          # 🆕 已在穿的技巧
      - recency(-15)                     # 最近推荐过
      - mastered(-8)                     # 已掌握
      - disliked(-20)                    # 用户不喜欢
      - feedback_adjustment(0~-10)       # 🆕 反馈驱动调整
```

新增评分方法：
- `_body_modifier_bonus()` — 读取 shoulder_hip_ratio、leg_body_ratio、height
- `_style_direction_bonus()` — 读取 preferred_style + 技巧的 style_variants
- `_style_anti_synergy()` — 风格反模式触发负分（如 City Boy × full_tuck = -5）
- `_aesthetics_synergy_bonus()` — Checklist 弱点 × 技巧 aesthetics_dimensions 协同
- `_weather_relevance_bonus()` — 温度/季节 × 技巧 seasons

**`core/suggestion.py`** — 建议引擎
- `SuggestionEngine` 类，串联 SkillMatcher + LLM 建议生成
- `_build_user_message()` 中加入美学原理上下文
- `apply_hard_filters()` 硬过滤（浅色裤禁露踝、短袖衬衫禁卷袖等）

### Phase 4：内容互审系统

**`core/content_reviewer.py`** — 多模型互审引擎

```
技巧内容 (DeepSeek V4 生成)
      │
      ├──→ 审稿人 A (MiniMax-M3) → 评审报告
      │
      └──→ 审稿人 B (豆包 2.0 Lite) → 评审报告
                │
                └──→ 一致性判定
                      ├─ PASS: 双方都通过
                      ├─ FLAG: 一方不通过
                      └─ FAIL: 双方不通过
```

8 项评审维度：
1. 事实正确性
2. 内部一致性
3. 理论对齐
4. 安全性
5. 风格变体质量
6. 可操作性
7. 美学链接完整性
8. 内容完整性

**`prompts/content_review.md`** — 审稿人 system prompt

**`tools/review_skills.py`** — CLI 批量审查工具

每次 SkillMatcher 加载自动跑内容审查，结果：16 pass / 0 flag / 0 fail。

## 踩坑记录

### 坑 1：`skill_id` vs `id` 字段名
- **现象**：遍历技巧时报 `KeyError: 'id'`
- **原因**：所有技巧 JSON 使用 `skill_id`，测试脚本用了 `s['id']`
- **修复**：统一使用 `skill_id`

### 坑 2：`task_routing` vs `routing` 属性名
- **现象**：检查代码时代码补全提示 `task_routing`
- **原因**：AIRouter 内部属性是 `self.routing`，非 `self.task_routing`
- **影响**：仅影响代码检查，不影响实际运行

### 坑 3：效果图 URL 未保存到结果
- **现象**：对比测试结果中只有图片数量，没有 URL
- **原因**：测试脚本只保存 `effect_images`（计数），未保存实际 URL
- **修复**：创建独立下载脚本，将效果图保存到 `data/effect_output/`

### 坑 4：Read 工具无法显示 PNG
- **现象**：Read 工具读取 PNG 显示 `[Unsupported Image]`
- **原因**：终端环境不支持内嵌图片显示
- **修复**：用 macOS `open` 命令启动系统图片查看器
